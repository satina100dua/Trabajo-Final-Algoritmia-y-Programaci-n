## 3. Nombre del proyecto y detalles
El proyecto consiste en realizar el programa de un parqueadero, que prestara su servicio a los vehículos del sector de la Universidad de Antioquia. De esta manera, denominamos al parqueadero "La Guardería" el cúal solo permite el ingreso de automóviles, el ingreso de motos no es aceptado. El programa permitirá registrar usuarios, ingresar y retirar vehículos y generar reportes administrativos. Adicionalmente, el parqueadero"La Guardería" cuenta con cuatro personas que trabajan turnos de 6 horas para atender a los usuarios que ingresan a usar el servicio y no presentar interrupciones ni contratiempos en el horario de 6:00 a.m. a 12:00 p.m. en jornada continua.

## 5. Reporte de visión
El proyecto consiste en el desarrollo de un sistema para el funcionamiento de un parqueadero, utilizando el lenguaje de programación Python y el entorno de desarrollo Spyder. El sistema está diseñado para controlar los ingresos y salidas de vehículos, así como también, para calcular automáticamente la duración en el parqueadero y el respectivo valor a pagar. De igual manera, podrá monitorear la disponibilidad de espacios en tiempo real.

El objetivo del proyecto es crear un sistema que permita controlar el ingreso y salida de vehículos en el parqueadero, gestionar el tiempo de permanencia y generar automaticamente reportes. Este sistema, desarrollado en Spyder, optimizará los procesos y permitirá una mejor gestión en el parqueadero "La Guardería".

Los beneficios que buscamos con este proyecto son automatizar la gestión del parqueadero, desde el registro de ingreso y salida de vehículos, lo que permitirá eliminar procesos manuales, reducir errores y agilizar las operaciones. Con la implementación del sistema, también se pretende reducir el tiempo de espera para los usuarios, haciendo que el proceso de ingreso y salida sea más rápido y eficiente. Esto contribuirá a mejorar la experiencia de los usuarios, ofreciendo un servicio más óptimo y rápido, lo que incrementará la confianza y permanencia de los usuarios al parqueadero.

## 6. Especificación de requisitos
A continuación se presentan los requisitos funcionales y no funcionales del software

**Funcionales**
- El sistema permitirá registrar el ingreso de vehículos.
- El sistema permitirá registrar la salida de vehículos.
- El sistema permitirá calcular el tiempo de permanencia en el parqueadero "La Guardería"
- El sistema podrá calcular el valor del servicio, teniendo en cuenta el tiempo de permanencia en el parqueadero (por hora y fracción)
- El sistema deberá generar y mostrar un recibo al momento de la salida, en el que se indique la hora de entrada, la hora de salida salida y el valor total.
- El sistema tendrá la capacidad de llevar un registro de los vehículos que se encuentran en el parqueadero y así mismo, de los espacios disponibles (máximo 64).
- El sistema permitirá generar reportes administrativos diarios, en los cuales se incluyan el número de vehículos y el total recaudado durante la jornada.
- El sistema deberá tener la capacidad de manejar errores que se presenten, como por ejemplo intentar ingresar un vehículo cuando el parqueadero no tiene espacio.
- El sistema debe exportar los reportes a un archivo CSV
  
**No funcionales**
- El programa debe funcionar correctamente en Spyder usando Python.
- Cada accion del sistema debe responder lo más rápido posible.
- La consola debe ser fácil de entender y utlizar para cada usuarios.
- Los datos del parqueadero deben quedar bien almacenados para evitar pérdidas en el programa.
- Todo el código debe guardarse en GitHub y el lider debe ser el encargado de administrarlo.

## 8. Plan de versionado
Para el desarrollo del sistema de gestión del parqueadero “La Guardería”, se utilizó como entorno de programación el software Spyder 6.0.0 (Scientific Python Development Environment), una herramienta potente y especializada para desarrollo en Python, orientada a tareas de análisis, educación, ingeniería y automatización. Esta versión, fue lanzada oficialmente el 3 de septiembre de 2024.

Spyder 6 introdujo por primera vez conexión con entornos remotos vía SSH, facilitando la ejecución del código en servidores externos sin salir del entorno local. En este proyecto, Spyder 6 se utilizó para organizar el código modular en archivos separados (menu.py, Facturas.py, registro_usuario2.py, entre otros), realizar pruebas interactivas en la consola IPython, y depurar errores mediante su sistema de explorador de variables y terminal. Todas estas características ofrecieron una experiencia de desarrollo fluida, clara y adecuada para el proyecto en base a buenas prácticas de programación. 

A continuación, describiremos el avance de cada procedimiento realizado desde el inicio hasta la entrega final. 

- Inicio del proyecto (primera reunión): Como grupo nos reunimos y se realizó una lectura de todo el documento para saber que teníamos que realizar para la primera entrega y se definieron los integrantes y el rol que íbamos a ejercer cada uno, se elaboraron las actas de entendimiento, colaboración y responsabilidad, también definimos el nombre del proyecto, el objetivo general, el nombre del repositorio y se construye el plan del proyecto, el cronograma (Gantt).
  
- Registro de usuario para el parqueadero (Segunda Reunión): Durante esta jornada de trabajo se implementó el módulo de registro de usuarios, el cual consistió en la creación de la función registro_usuario2 y la estructura del diccionario dicUsuarios que almacena los datos. En esta fase se programaron las validaciones para cada campo solicitado al usuario: se verificó que el nombre y el apellido no contuvieran números y tuvieran una longitud mínima de cuatro letras; el documento debía ser numérico y tener entre 4 y 15 dígitos; y la placa debía cumplir con el formato exacto de tres letras seguidas de tres números. Además, se implementó un control para evitar el registro duplicado de usuarios ya existentes. Finalmente, se validó que la función devolviera un diccionario actualizado solo si los datos ingresados eran correctos, mostrando mensajes informativos o de error según el caso.
  
- Ingreso de vehículos (Tercera reunión): Se desarrolló el módulo de ingreso de vehículos al parqueadero, el cual se conectó directamente con el registro previo de usuarios. Se programó una validación que verifica que el documento ingresado corresponda a un usuario ya registrado y que la placa del vehículo no esté actualmente dentro del sistema para evitar duplicados. Al cumplir estas condiciones, se registran automáticamente la fecha y hora exactas de ingreso. Estos datos se almacenan en el diccionario dicRegistro, bajo la clave de la placa. Adicionalmente, se integró la función ImprimirFacturaIngreso, que genera en consola un comprobante visual del ingreso con los datos del cliente, el vehículo y la hora registrada, simulando una factura.
  
- Retiro de vehículos y cálculo de cobros (Cuarta Reunión): Durante esta fase del proyecto se desarrolló el módulo de retiro de vehículos, integrando la lógica para calcular el tiempo de permanencia y el monto a pagar por el uso del parqueadero. A partir del tiempo de ingreso previamente almacenado en dicRegistro, se registró el timestamp de salida y se calculó la diferencia en segundos. Con esto se implementó la función CalculoPago, que convierte este tiempo en horas completas y cuartos de hora, aplicando la tarifa definida. Se incluyó una condición que establece un pago mínimo de una hora, aunque el tiempo sea menor. Posteriormente, se generó una factura detallada usando la función ImprimirFactura, mostrando el nombre del cliente, la placa, el tiempo facturado y el valor total.
  
- Módulo de administración (Quinta reunión): En esta etapa se desarrolló el módulo de administración del sistema, al cual solo pueden acceder usuarios autenticados con nombre de usuario y contraseña. Se programaron funciones para mostrar el total de vehículos registrados, vehículos retirados, vehículos que aún no han salido, el total recaudado en cobros, el tiempo promedio de permanencia y los vehículos con el mayor y menor tiempo de parqueo. Estos datos se obtienen directamente de las listas vehiculos_retirados y dicRegistro. Para los cálculos de tiempo, se implementó una función que transforma los segundos en horas, minutos y segundos, ofreciendo al administrador reportes precisos y legibles.
